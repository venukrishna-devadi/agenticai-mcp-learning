═══════════════════════════════════════════════════════════════════════════════

    Step 2: search() → no result
    Step 3: search() → no result
    Step 4: search() → no result
    Step 5: search() → no result
    Step 6: need_tool = false, force exit with "Max steps reached"


# PROJECT 3.4 - Iterative ReAct Agent

## Key Concepts:
1. **ReAct = Reasoning + Acting** (from the paper)
2. **Loop, not linear** - agent can think multiple times
3. **Working memory** - history field stores past steps
4. **Loop guards** - prevent infinite loops (MAX_STEPS)
5. **Dynamic decision making** - agent decides when to stop

## Three Node Types:
- **Think Node**: Decides what to do next (JSON output)
- **Tool Node**: Executes the action (calculator, search)
- **Final Node**: Synthesizes final answer when done

## The Loop:
Think → Tool → Observe → Think → Tool → Observe → ... → Final

## Critical State Additions:
- `step_count`: Prevents infinite loops
- `history`: Working memory across cycles

## Why This Matters:
- Foundation for ChatGPT, Claude, AutoGPT
- Enables multi-step reasoning
- Allows tool chaining
- Makes agents truly autonomous


LLM + MEMORY + TOOLS + REASONING LOOP + STATE MACHINE
That combination is: modern agent architecture.

LLMs are probabilistic reasoners, not deterministic calculators.
Tools fix this.
This is one of the MOST important lessons in Agentic AI.
